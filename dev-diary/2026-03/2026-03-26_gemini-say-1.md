# Chiedi a Gemini del 26 marzo 2026

## Domanda 1

Questo branch ha laravel 12 e deo passare a laravel 13.
Qual'è la check list

## Risposta 1

Per passare da Laravel 12 a Laravel 13 nel tuo progetto yaPCP, basandomi sui tuoi diari degli ultimi giorni e sulle problematiche riscontrate (specialmente con LaRecipe), ecco la checklist operativa che dovresti seguire.

### 1. Preparazione e Requisiti

- [x] PHP Version: Assicurati di utilizzare PHP 8.4 o superiore (i tuoi file AGENTS.md indicano già PHP 8.5 come target).
- [x] Backup: Hai già creato il branch feat/113-laravel-12-to-13, il che è perfetto. Assicurati che il tuo ambiente locale sia pulito.

### 2. Aggiornamento delle dipendenze (composer.json)

Modifica il file composer.json aggiornando le versioni dei pacchetti core:

- [ ] "laravel/framework": "^13.0"
- [ ] "laravel/breeze": "^2.x" (o la versione compatibile con L13)
- [ ] "pestphp/pest": "^4.x"
- [ ] Controlla gli altri pacchetti laravel/* (Sail, Pint, Pail, Prompts).

### 3. Gestione del blocco LaRecipe

Come hai notato nel diario del 23/03 e 26/03, LaRecipe causa un KO.

- [ ] Patch LaRecipe: Non aspettare necessariamente l'aggiornamento ufficiale se hai urgenza. Segui il suggerimento del gruppo Telegram e punta alla PR specifica o modifica temporaneamente il tuo composer.json per accettare la versione patchata:

```json
"binarytorch/larecipe": "dev-master as 2.5.0" 
```

_(Oppure applica le modifiche indicate nella PR #416 se stai usando un fork o un repository locale)._

### 4. Esecuzione dell'aggiornamento

- [ ] Esegui composer update.
- [ ] Se incontri conflitti, prova a usare composer update --with-all-dependencies per permettere a Composer di risolvere i sotto-pacchetti.

### 5. Verifica dei file di sistema (Laravel 13)

Laravel 13 continua l'evoluzione della struttura snella introdotta con la 11:

- [ ] bootstrap/app.php: Controlla se ci sono nuovi metodi di configurazione per middleware o eccezioni.
- [ ] Config Files: Se hai file di configurazione pubblicati, confrontali con quelli di default di L13 (puoi usare php artisan config:publish --all o controllare il repository ufficiale).

### 6. Test e Validazione (Fondamentale)

Come suggerito da Gemini il 21/03:

- [ ] Test Livewire 3: Verifica che l'attuale stack Livewire 3 funzioni correttamente su Laravel 13 prima di passare alla versione 4 (che hai nel branch feat/112).
- [ ] Pest Suite: Esegui tutti i test:

```bash
php artisan test --compact
```

- [ ] Browser Check: Naviga nelle sezioni critiche (Dashboard, Contest List) per escludere errori di rendering dovuti a modifiche nel motore Blade o nelle macro.

###  7. Pulizia e Stile

- [ ] Esegui Pint per assicurarti che il codice rispetti i nuovi standard:

```bash
bash
vendor/bin/pint --format agent
```

Prossimo step suggerito dai tuoi diari: Una volta stabilizzato Laravel 13 con Livewire 3, potrai fare il merge del branch feat/112-livewire-3-to-4 per completare l'aggiornamento dello stack tecnologico.
