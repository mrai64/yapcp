To manage the project that grow I try to follow 2 standard using
github project to manage TDL | To Do List

Prioritize w/ABC system, using (A) (B) (C) (D) (E) after the conventional commits
see official definition
in few words (A) is the more urgent Must Do, (B) Should Do, (C) Nice to Do, (D) Delegate, (E) Eliminate from the list
Follow Conventional commits, using feat: fix: chore: refactor:
see official definition
FIFO | First In First Out After the conventional commits and ABCDE codes i usually add an ID
composed by iso-date yyyy-mm-dd followed by a counter-in-day that don' start with 0 but with 1. Supposing to have in a day more than 9 issues|ideas to track, every counter is 2 digit long. [id:yyyy-mm-dd.nn]
The ID can be put in the end of issue title, and for project panel replicated in the open id column.

Before closing an issue the prioritize code should be removed. Should be.

Issue titles that don't follow these few rules must be modified to conform.

A branch for every Issue following a rule for branch naming:
branch name must be preceded by pa conventional commits feat/ fix/ chore/ refactor/
and must start with issue id, i.e. feat/100-agreements. Using LaRecipe package
to write manuals, one is reserved to developers and collect an md file for
every branch name i.e. feat/100-agreements.md
